## 1. 압축 내용

K-뷰티에 관심 많은 필리핀 20-30대 여성이 현지보다 저렴한 가격에 한국 화장품을 구매할 수 있는 원스톱 랜딩 페이지

---

## 2. 핵심 내용

**해결할 문제**
- 필리핀 현지 K-뷰티 제품 가격이 한국 판매가 대비 상당히 높음
- 낱개 구매 시 구매대행 이용하면 배송료로 인해 오히려 더 비싸짐
- 구매대행 이용 시 배송 기간이 길어 제품을 빠르게 받기 어려움

**제안 솔루션**
- 한국에서 사전 구매한 제품을 필리핀 현지 파트너에게 배송하여 현지 재고로 확보
- 고객이 랜딩 페이지에서 주문하면 당일/익일 배송으로 빠른 수령 가능
- 배송 후 운송장 번호를 이메일로 제공하여 배송 조회 가능

**예상 임팩트**
- 랜딩 페이지 운영 후 2주 이내 10개 판매 목표 달성
- 추가 판매 시 알림 받고 싶다는 고객 2명 이상 확보 (재구매 관심 지표)
- 구매-배송-피드백 운영 프로세스 검증 및 개선점 수집
- 어필리에이트 마케팅으로 지속 협업 가능한 인플루언서 2명 이상 확보 (판매당 이윤의 20% 지급)

### Opportunity

**제품 전략**
- 초기 제품: 조선미녀 데일리 틴티드 선세럼 (필리핀 K-뷰티 인플루언서 피드백에 따라 변경 가능)
- 제품 선정 기준: 올리브영에서 구매 가능한 필리핀 인기 한국 브랜드의 신제품
- 신제품 타겟 이유: 할인 가능성 + 현지 구매처 희소성

**시장 분석**
- 필리핀 K-뷰티 시장은 성장 중 (정확한 시장 규모 수치 미확보)
- 타겟 고객: K-뷰티 인플루언서 콘텐츠를 적극 소비하는 관심층 여성

**경쟁 환경**
- 주요 경쟁: Shopee.ph 및 현지 K-뷰티 오프라인 매장
- 경쟁 전략: 소규모 MVP로 수익보다 학습 우선, 이윤 낮춰 가격 경쟁력 확보 (Shopee와 유사하거나 약간 우위)

### Goals & Non-Goals

**Goals**
- 2주 내 10개 판매 달성
- 재구매 관심 고객 2명 이상 확보
- 운영 프로세스 검증 및 피드백 수집
- 어필리에이트 마케팅으로 지속 협업 가능한 인플루언서 2명 이상 확보 (판매당 이윤의 20% 지급)

**Non-Goals**
- 다양한 제품 라인업 구축 (단일 제품 집중)
- 회원가입/로그인 기능
- 장바구니 기능
- 정기 구독 기능
- 복잡한 회원 등급/포인트 시스템
- 다중 결제 수단 (PayPal만 지원)

#### Metrics

| 지표 | 목표 | 측정 시점 |
|------|------|-----------|
| 총 판매 건수 | 10개 | 2주 |
| 재구매 관심 고객 수 | 2명 | 2주 |
| 협업 인플루언서 확보 | 2명 | 2주 |

#### Measurement Method

- **판매 건수**: 주문 데이터 자동 집계 (문제 발생 시 수동 조정)
- **재구매 관심**: 구매 고객에게 이메일 발송 → 관심 표시 클릭 시 데이터 시트에 자동 저장
- **인플루언서 성과**: 어필리에이트 코드별 판매 자동 추적 (코드 파라미터로 유입 경로 식별)

### Constraints

**기술적 제약**
- 결제 시스템: PayPal 사용 (필리핀 고객 결제 지원 필수)
- 호스팅/인프라: Cloudflare 무료 플랜 우선 (Google Cloud 무료 크레딧 보조적 고려)
- 트래픽/용량: 무료 플랜 범위 내 (초기 소규모 트래픽 가정)

**비즈니스 제약**
- 총 예산: 20만원 (제품 구매 + 한국→필리핀 배송료 포함)
- 환불 정책: 배송 중 문제로 상품 불량 시에만 환불 가능 (3일 이내 요청 + 미사용 상태 + 상품 반송 후 환불 처리)
- ✨(추가) 주문 취소 정책:
  - 배송 시작 전: 전액 환불 (PayPal 결제 수수료 제외)
  - 배송 시작 후: 배송비 차감 후 환불 (PayPal 결제 수수료 제외)
  - 취소 시 재고 자동 복구

**규제 제약**
- 필리핀 전자상거래 및 화장품 판매 관련 규제: 현재 미확인 상태 (런칭 전 확인 필요)
- 개인정보 보호: 준수 필요 (구체적 요구사항 추후 확인)

#### Violation Actions

- 3일 경과 또는 사용 흔적 확인 시 환불 거부
- 상품 미반송 시 환불 처리 보류

### Users & Scenarios

**Primary Users**

**1. 신제품 얼리어답터 - 20-30대 여성**
- **목표**: 신뢰하는 K-뷰티 브랜드의 신제품을 저렴한 가격에 빠르게 구매
- **페인 포인트**: 신제품 구매처 부족, 현지 가격 부담, 배송 불확실성
- **시나리오**: SNS에서 인플루언서가 소개한 신제품 포스팅 확인 → 어필리에이트 링크로 랜딩 페이지 접속 → 상대적으로 저렴한 가격과 빠른 배송(현지 택배사), 실시간 추적 가능 확인 → 구매 결정

**2. 마이크로 인플루언서 - 팔로워 1K-10K**
- **목표**: 수익 창출 + 신제품 정보 제공으로 팔로워 만족도 향상
- **페인 포인트**: 신뢰할 수 있는 협업 파트너 부족, 저렴하면서도 빠른 배송 제공처 찾기 어려움
- **시나리오**: 지인 소개 또는 직접 검색으로 협업 제안 DM 수신 → 건당 이윤 20% 조건 + 현지 배송 파트너 보유 + 지속 협업 가능성 확인 → 신뢰감 형성 → 탐색적 첫 협업 시작

### Requirements (MoSCoW)

#### Must Have (M)

**Functional**
- REQ-F01: 고객은 제품 상세 정보(이미지, 설명, 가격)를 확인할 수 있어야 함
- REQ-F02: 고객은 PayPal로 결제할 수 있어야 함
- REQ-F03: 고객은 배송지 정보(이름, 주소, 연락처, 이메일)를 입력할 수 있어야 함
- REQ-F04: 고객은 배송지 입력 시 주소 자동완성을 이용할 수 있어야 함 (무료 플랜 범위 내 외부 API 사용)
- REQ-F05: 인플루언서 어필리에이트 코드가 URL 파라미터로 추적되어야 함
- REQ-F06: 어필리에이트 코드별 판매 데이터가 기록되어야 함
- REQ-F07: 인플루언서는 `/affiliate/stats/{code}` URL로 자신의 성과를 조회할 수 있어야 함 (로그인 불필요)
- REQ-F08: 대시보드는 총 클릭 수, 총 판매 건수, 누적 수익, 대기 중 수수료를 표시해야 함
- REQ-F09: 구매 완료 시 주문 확인 이메일에 주문 조회 링크가 포함되어야 함
- REQ-F10: 배송 완료 시 운송장 번호가 이메일로 발송되어야 함
- REQ-F11: 재구매 관심 확인 이메일이 발송되고, 클릭 시 데이터 시트에 기록되어야 함
- REQ-F12: 페이지는 영어와 필리핀어(Tagalog) 토글로 제공되어야 함
- REQ-F13: 고객은 문의 버튼을 통해 문의할 수 있어야 함 (이메일 또는 폼)
- REQ-F14: 재고는 자동으로 관리되어야 하며, 필요 시 수동 조정 가능해야 함
- REQ-F15: 주문 데이터는 자동으로 집계되어야 하며, 필요 시 수동 조정 가능해야 함
- ✨(추가) REQ-F25: 고객은 주문 상세 페이지에서 주문을 취소할 수 있어야 함
- ✨(추가) REQ-F26: 배송 시작 전 취소 시 전액 환불되어야 함 (PayPal 수수료 제외)
- ✨(추가) REQ-F27: 배송 시작 후 취소 시 배송비를 차감한 금액이 환불되어야 함 (PayPal 수수료 제외)
- ✨(추가) REQ-F28: 취소 완료 시 확인 이메일이 발송되어야 함
- ✨(추가) REQ-F29: 취소된 주문의 재고는 자동으로 복구되어야 함

**Non-Functional**
- REQ-NF01: 페이지 로딩 속도는 3초 이내여야 함 (모바일 3G 기준)
- REQ-NF02: 개인정보(이름, 주소, 이메일)는 암호화되어 저장되어야 함
- REQ-NF03: PayPal 결제는 HTTPS로 보안 처리되어야 함

#### Should Have (S)

- REQ-F16: 페이지에 한국 실사용자 후기(Static)가 표시되어야 함 (출처 명시: "한국 사용자 후기")
- REQ-F17: 현재 재고 수량이 표시되어야 함 (구매 긴급성 유도)
- REQ-F18: FAQ 섹션이 제공되어야 함

#### Could Have (C)

- REQ-F19: 소셜 미디어(Facebook, Instagram) 공유 버튼이 제공될 수 있음 (기본 링크 방식)

#### Won't Have (W)

- REQ-F20: 회원가입/로그인 기능은 제공하지 않음 (비회원 구매만)
- REQ-F21: 장바구니 기능은 제공하지 않음 (단일 제품 즉시 구매)
- REQ-F22: 정기 구독 기능은 제공하지 않음
- REQ-F23: 다중 결제 수단은 제공하지 않음 (PayPal만)
- REQ-F24: 고객 포인트/쿠폰 시스템은 제공하지 않음

### Dependencies & Risks

**Dependencies**
- PayPal 계정 개설 및 결제 통합 → **Critical Path**
- 필리핀 현지 파트너 확보 및 배송 프로세스 합의 → **Critical Path**
- 주소 자동완성 API (무료) 선정 및 통합
- 제품 조달 (올리브영 구매)
- ✨(추가) PayPal Refund API 통합 (환불 처리) → **Critical Path**

**Risks**

| 리스크 | 영향도 | 대응 방안 |
|--------|--------|----------|
| 필리핀 규제 미준수로 법적 문제 발생 | High | 런칭 전 필리핀 전자상거래/화장품 판매 규제 확인 |
| 현지 파트너 배송 지연/문제 | High | 파트너와 배송 처리 기한 명확히 합의 (예: 주문 후 24시간 내 배송) |
| 제품 배송 중 파손 | Medium | 포장 강화 요청, 파손 발생 시 환불 정책 적용 |
| 동시 주문으로 재고 초과 판매 | Medium | 결제 전 재고 확인 로직 + 재고 부족 시 자동 품절 처리 또는 대기 리스트 안내 |
| ✨(추가) PayPal 환불 처리 실패로 고객 불만 | High | 환불 실패 시 자동 재시도 + 수동 처리 프로세스 준비 |
| ✨(추가) 취소 후 재고 복구 실패로 재고 불일치 | Medium | 트랜잭션으로 취소-환불-재고 복구를 원자적으로 처리 |
| ✨(추가) 어필리에이트 커미션 지급 후 주문 취소 시 손실 | High | MVP에서는 커미션 회수 없이 손실 감수, 향후 인플루언서와 정산 프로세스 협의 필요 |

## 작성 원칙

1. **정량화**: 모든 목표와 요구사항은 측정 가능한 수치로 표현
2. **원자성**: 한 요구사항 = 한 기능/행위/조건
3. **간결성**: 불릿/표 활용, 장황한 설명 지양
4. **추적성**: 모든 REQ는 Epic와 연결 (Epic Overview 작성 후 Traceability 섹션 업데이트)
5. **What/Why만**: How(구현 방법)는 Tech Spec으로 분리
6. **MECE 원칙**: 요구사항은 상호 배타적이고 완전히 포괄적으로
7. **Non-Goals 명시**: 스코프 크리프 방지를 위해 명확히 제외 항목 정의
8. **범위 명확화**: 제품 전체 vs 특정 기능을 문서 시작 시 명시
9. **상세도 조절**: 제품 전체는 높은 수준, 특정 기능은 구체적으로
