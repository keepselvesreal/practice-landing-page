# 문서 버전 관리 자동화 프롬프트

아래 작업 흐름에 따라 문서 버전 관리 작업을 수행해줘.

## 작업 흐름

### 1. 파일 정보 확인
사용자에게 질문:
- 새로운 파일 경로 (수정 내용 반영됨)
- 기존 파일 경로 (타임스탬프 포함, 예: `filename--25-10-28-2109.md`)
- 기존 파일 이동 디렉토리 (예: `docs/landing_page/inbox/`)

### 2. 기존 파일 처리
1. 메타데이터 읽기
2. 메타데이터 업데이트:
   - `status`: `Superseded`
   - `links.out`: 새 파일 절대 경로 추가
   - `links.in`: 빈 배열 유지
3. 지정된 디렉토리로 이동 ⭐ (이 작업을 먼저 해야 함)

### 3. 새 파일 처리
기존 파일의 메타데이터 복사 후 업데이트:
- `created_at`: 현재 시간 (`date` 명령어)
- `version`: 기존 version + 1
- `status`: `Active`
- `links.in`: 이동된 후의 기존 파일 절대 경로 (배열)
- `links.out`: 빈 배열 또는 기존 out 유지

## 핵심 원칙

**작업 순서**: 기존 파일 이동 → 새 파일 업데이트
- 이동 후 경로가 새 파일의 `links.in`에 올바르게 반영됨

**필드 의미**:
- `links.in`: 현재 파일을 참조하는 파일 경로
- `links.out`: 현재 파일이 참조하는 파일 경로
