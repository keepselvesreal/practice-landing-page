# 11장: 마무리 - 추출된 정보

## 핵심 내용
- 효과적 테스트 프로세스는 순차가 아니라 반복적이며, 상황에 따라 기법 순서를 유연하게 바꾼다.
- 모든 기법을 적용해도 소프트웨어가 완벽해지진 않으며, 도메인 지식과 사용자 관점 확인이 필수다.
- 자동화된 “지능형 테스트”와 모니터링은 미래 테스트 전략의 중요한 축으로 부상한다.
- 현실에서는 단위 테스트가 어렵기 때문에, 레거시 시스템과의 경계에서 새로운 코드에 테스트 가능 구조를 도입해야 한다.
- 품질 보증은 배포 이후 모니터링, 사용자 참여, 조직 문화와 결합돼야 지속적인 개선이 가능하다.

## 상세 핵심 내용

### 반복적 테스트 흐름
- 명세 기반, 구조 기반, 경계, 속성, 더블, 대규모 테스트 등은 필요에 따라 왕복하며 적용한다.
- TDD/설계/테스트 기법을 조합해 기능 구현과 검증이 자연스럽게 섞인 개발 루프를 만든다.

### 버그 제로 신화와 기대 관리
- 다양한 관점에서 테스트해도 복잡한 상호작용의 모든 결함을 잡을 수는 없음.
- 도메인 이해와 팀 학습이 결함 예방에 결정적이며, 잔여 위험을 인지하고 대응 전략을 마련해야 한다.

### 지능형 테스트와 자동화
- Sapienz, 대규모 퍼징 등 자동화 탐색 도구가 점점 실용화되고 있으며, EvoSuite 같은 도구로 실험을 권장.
- 인간은 테스트 설계, 도구는 대규모 입력 탐색/실행을 담당하는 협업 모델 제안.

### 사용자 관점 포함
- 검증(verification)뿐 아니라 사용자가 원하는 기능인지 확인하는 validation 활동과의 연계 필요.
- 사용자 피드백, 실사용 데이터, 실험(AB 테스트 등)을 통해 제품 가치 검증.

### 레거시 코드 전략과 현실 조언
- 테스트하기 어려운 기존 클래스는 새로운 테스트 가능 클래스로 감싸고, 경계에서 호출하도록 설계.
- 점진적 리팩터링, 새로운 코드에 TDD/테스트 가능성 적용, 잔존 레거시는 감싸기 패턴 사용.

### 운영과 품질
- 배포 후 모니터링, 로깅, 경보 체계를 갖춰 테스트가 놓친 문제를 빠르게 감지.
- 테스트 결과, 모니터링 데이터, 사용자 피드백을 통합해 지속적 개선 사이클 구축.

## 주요 화제

### 1. 반복적 테스트 모델 재확인
- 다양한 기법을 상황에 따라 왕복 적용.

### 2. 버그 제로의 한계
- 현실적 기대 설정, 도메인 지식 중요성.

### 3. 지능형 테스트 전망
- 자동 테스트 생성, 탐색적 도구, 인간-기계 협업.

### 4. 사용자·운영 관점 결합
- Validation, 모니터링, 피드백 루프 구축.

## 부차 화제

### 1. 조직 문화
- 테스트 기법을 팀 표준으로 내재화하고, 지속 학습 문화 마련.

### 2. 도구 추천
- EvoSuite, Sapienz, Google 퍼징 등 연구/산업 사례.

### 3. 레거시 통합 패턴
- 새 코드 추출, 래퍼/어댑터 사용으로 테스트 가능성 확보.

### 4. 모니터링과 품질 지표
- 배포 후 지표, 로그, 알림 체계의 중요성.

### 5. 테스트 철학 재정리
- “테스트는 개발자 책임”이라는 메시지와 책 전반 핵심 요약.
