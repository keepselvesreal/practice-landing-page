# 8장: 테스트 주도 개발 (TDD) - 추출된 정보

## 핵심 내용
- TDD는 “테스트 작성 → 실패 확인 → 최소 구현 → 리팩터링”의 짧은 루프를 반복해 기능과 설계를 동시에 다듬는다.
- 작은 예제부터 시작해 점진적으로 일반화하며, 테스트가 설계 결정을 이끄는 역할을 한다.
- 빨리 실패를 확인하고 즉시 피드백을 얻어 회귀를 줄이며, 리팩터링이 안전해진다.
- 모든 상황에 TDD를 강제할 필요는 없으며, 복잡한 UI/탐색적 프로토타입 등에는 다른 전략이 유효하다.
- 숙련도를 높이기 위한 훈련법(Kata, Baby Steps, Mockist vs Classicist 등)과 조직 도입 팁을 제시한다.

## 상세 핵심 내용

### TDD 루프 실습
- 로마 숫자 변환 예제로 `Red → Green → Refactor` 과정을 단계별 시연.
- 케이스 목록을 먼저 나열하고 가장 단순한 사례부터 테스트를 작성해 구현 확장을 유도.
- 리팩터링 단계에서 중복 제거, 데이터 구조 도입(Map), 메서드 추출 등을 수행.

### 생산성·설계 향상 효과
- 테스트가 곧 명세이자 예제로 기능 요구를 명확히 문서화한다.
- 즉각적인 피드백으로 디버깅 시간 감소, 결함 발견 시점이 앞당겨진다.
- 리팩터링 시 기존 테스트가 안전망 역할을 해 구조 개선이 쉬워진다.
- 설계 측면에서 작은 단위로 생각하게 되어 응집도 상승, 결합도 감소.

### 실무 적용 전략
- 기존 코드 작업 시 “커버리지 확보 → 스몰 리팩터링 → TDD 루프로 신규 기능 추가” 패턴 활용.
- 팀/프로젝트 규칙: 한 번에 하나의 실패 테스트 유지, 5~10분 사이클 유지, 테스트와 구현 간 명확한 의도 표현.
- 도구(IDE 단축키, 테스트 러너)와 자동 실행 환경을 준비해 피드백 속도를 높인다.

### TDD 한계와 예외 상황
- UI/그래픽, 하드웨어 연동, 탐색적 분석 등에는 빠른 프로토타이핑 후 테스트 보강이 더 적합할 수 있음.
- 높은 불확실성 과제에서는 스파이크 코드를 작성한 뒤 TDD로 재구현하는 전략을 권장.
- 조직이 TDD를 강제할 경우 과도한 형식주의로 흐를 수 있으므로 실용적 적용 범위를 명확히 한다.

### 학습과 팀 확산
- 코딩 카타, 페어 프로그래밍, 모브 세션으로 루프 감각을 익힌다.
- Classicist(상태 검증 중심) vs Mockist(행동 검증 중심) 스타일 비교, 팀 상황에 맞게 선택.
- 레드-그린 전환을 강제하는 IDE 플러그인, 타이머 등으로 리듬을 유지.

## 상세 내용

### 리팩터링 단계 워크플로우
- 테스트 통과 후 중복 제거, 알고리즘 일반화, 데이터 구조 변경 등 개선을 수행.
- 리팩터링 중 실패가 발생하면 즉시 다음 녹색 상태로 되돌아가 안정 상태 유지.

### 테스트 품질 관리
- 테스트는 간결·명확·중복 최소 원칙을 따른다.
- 의도 표현을 위해 “Given-When-Then” 네이밍, 설명 변수 도입, 헬퍼 메서드 활용.

### 심리적 장점
- 실패를 허용하는 안전한 루프가 두려움을 줄여 실험을 장려.
- 지속적인 성공 체감이 개발자 몰입감을 높여 생산성 향상.

## 주요 화제

### 1. TDD 기본 사이클
- Red-Green-Refactor 단계, 예제 기반 구현 시연.

### 2. 설계 품질과 피드백
- TDD가 코드 구조 개선과 빠른 피드백 제공에 기여.

### 3. 실무 도입 전략
- 기존 코드 대응, 팀 운영 규칙, 도구 지원.

### 4. 한계와 예외 상황
- TDD 적용이 어려운 영역, 대체 전략, 현실적 기대치 설정.

## 부차 화제

### 1. TDD 스타일
- Classicist vs Mockist 비교, 각 접근의 장단점.

### 2. 연습 방법
- Kata, Randori, Baby Step 타이머 등 학습 루틴.

### 3. 테스트 명명·구조화
- Given/When/Then, Arrange-Act-Assert 패턴 제안.

### 4. 문화·조직 요소
- 팀 내 합의, 코드 리뷰에서 TDD 루프 검증, 지속 가능한 리듬 유지.

### 5. 도구
- IDE 자동 실행, 테스트 커버리지 시각화, 실패 시 알림 시스템 활용.
