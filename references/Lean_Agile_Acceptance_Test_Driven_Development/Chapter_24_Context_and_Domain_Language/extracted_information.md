# Chapter 24: Context and Domain Language - 추출된 정보

## 핵심 내용
- 협업 과정에서 공통 언어(ubiquitous language)를 정립하고 테스트·모델·코드 전반에서 일관되게 사용
- 용어 정의, 정체성/연속성 판단, 경계 컨텍스트 간 조율을 통해 오해를 줄이고 시스템 통합을 용이화
- 공통 도메인을 공유하는 시스템은 인터페이스 용어와 의미에 합의해야 하며, 테스트가 이를 검증

## 상세 핵심 내용

### 언어 정립
- 고객·개발자·테스터가 스토리, 테이블, 유스케이스 등에서 사용한 용어를 문서화하고 한 줄 정의 제공
- 고객 단위가 주도하되 모호하거나 중복된 용어는 개발·테스트가 피드백

### 엔터티 정의와 지속성
- 엔터티의 정체성(동일성 판단)과 지속성(얼마나 오래 존재해야 하는가)을 명확히 해 데이터 모델과 테스트가 일치하도록 함

### 경계 컨텍스트 관리
- 대기업 또는 복잡한 시스템은 여러 도메인이 공존; 공통 영역(예: 요금 체계)은 공유 언어를 합의하고 테스트 더블로 독립 검증
- 각 컨텍스트의 용어 차이를 인식하고 필요 시 번역 계층을 둠

## 주요 화제

### 테스트와 언어의 상호작용
- 테스트 표·컬럼명·시나리오 이름이 언어 정착에 기여하며, 변경 시 테스트가 빠른 피드백 제공

### 도메인 간 예시
- 항공 산업 사례로 용어(`flight`, `leg`) 의미 차이를 설명하며 공통 개념 정립의 중요성 강조

## 부차 화제

### 용어 정의 도구
- 표를 사용해 용어와 관계를 설명(예: Customer Type, Order Fields)하여 이해를 돕는 방법 제시

### 용어집 유지 관리
- 약어·용어는 용어집으로 관리하고 테스트·문서·코드 간 싱크를 맞추도록 권장
