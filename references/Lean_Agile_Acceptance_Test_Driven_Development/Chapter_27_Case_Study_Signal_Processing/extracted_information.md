# Chapter 27: Case Study – Signal Processing - 추출된 정보

## 핵심 내용
- 실시간 음향 신호 처리 시스템에서 인수 테스트를 설계하는 방법을 소개
- 캡처된 오디오 파일을 테스트 더블로 사용해 하드웨어 의존성을 줄이고 반복 가능성 확보
- 최종 Leq 계산뿐 아니라 중간 파이프라인 단계(윈도잉, A-가중)도 테스트 데이터로 검증

## 상세 핵심 내용

### 시스템 개요
- 마이크 입력 → A/D 변환 → 샘플 처리 → Leq 산출 흐름을 도식화
- 캘리브레이션 파일로 기준 레벨을 맞추고, 테스트 파일 대비 기대 Leq를 비교

### 테스트 데이터 전략
- 1초 길이(22,500 샘플)의 WAV/데이터 파일을 입력으로 사용, 기대 값은 전문 장비에서 측정한 결과를 오라클로 삼음
- 파일 기반 테스트로 시간/환경 변수 제거, 재현 가능한 자동화 지원

### 층별 테스트
- 최종 출력뿐 아니라 중간 산출물 파일을 제공해 개발자 테스트(유닛 수준)와 인수 테스트가 연계되도록 구성
- SME(신호 처리 전문가)가 테스트 데이터를 제공해 정확성과 신뢰성 향상

## 주요 화제

### 하드웨어 대체 기법
- 외부 장비 없이도 테스트 실행 가능하도록 캡처 파일을 활용하는 실무 팁

### 도메인 전문가 협업
- 전문가가 중간 단계 데이터까지 제공하면 개발자가 세밀한 회귀 테스트를 작성 가능

## 부차 화제

### 복잡 데이터 검증
- 인수 테스트는 단일 수치뿐 아니라 데이터 세트를 다룰 수 있음을 시사
