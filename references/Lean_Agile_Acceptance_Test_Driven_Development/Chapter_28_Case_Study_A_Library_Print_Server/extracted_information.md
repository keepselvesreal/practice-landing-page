# Chapter 28: Case Study – A Library Print Server - 추출된 정보

## 핵심 내용
- 도서관 프린트 서버의 전체 워크플로(문서 제출→큐 누적→릴리즈 스테이션 출력)를 테스트로 모델링
- 사용자의 잔액, 요금표, 문서 속성 등 초기 상태를 명확히 정의해 복합 시나리오를 재현
- 각 단계에서 시스템 응답(요금 승인, 큐 업데이트, 계정 차감, 출력 완료)을 검증하여 엔드-투-엔드 일관성 확보

## 상세 핵심 내용

### 셋업과 요금 정책
- 사용자 계정과 잔액, 흑백/컬러 요금 테이블, 출력할 문서 목록, 큐 초기 상태, 다음 작업 번호 등을 표로 정의

### 동작 흐름
- 문서 제출 시 요금 승인 메시지와 작업 번호 할당을 확인
- 릴리즈 스테이션 로그인 후 큐 목록 표시, 선택 출력, 계정 차감, 인쇄 완료 상태 등 연속 동작 검증

### 예외/변형 고려
- 인쇄 모드 차이, 여러 문서 처리, 잔액 부족 등 추가 케이스를 위해 별도 테스트가 필요함을 언급

## 주요 화제

### 워크플로 테스트 설계
- 다수의 유스케이스를 연결해 전체 경험을 검증하는 테스트 구조 예시 제공

### 데이터 일관성 확인
- 큐 상태, 계정 잔액, 출력 기록이 동기화되는지 단계별로 확인하여 회귀 시 신뢰성 확보

## 부차 화제

### 재사용 가능한 구성 요소
- 요금 계산, 계정 충전 등 하위 기능 테스트와 연계 가능성을 시사
