# Chapter 5: The Example Project - 추출된 정보

## 핵심 내용
- 프로젝트 개시 시점에서 작성하는 차터(비전·미션·목표·원칙)의 역할과 SMART 기준을 소개
- 차터 목표를 프로젝트 수준의 인수 테스트로 간주하여 측정과 검증 계획을 수립
- 고수준 요구 수집 워크숍을 통해 기능 후보를 발굴·정제하고 범위를 결정하는 과정을 설명
- 선정된 기능에 대한 상위 인수 기준을 정의하고 우선순위·일정 계획으로 연결

## 상세 핵심 내용

### 차터 구성요소
- 비전: 프로젝트가 달성하려는 미래상과 비즈니스 가치
- 미션: 비전을 달성하기 위한 접근 방식, 예: 맞춤형 소프트웨어 구축
- 목표: SMART(구체·측정·달성 가능·관련성·시간 제한) 기준으로 정의하여 성공 조건을 명확화
- 원칙: 의사결정 시 우선순위를 안내하는 가치 선언

### 프로젝트 단위 인수 테스트
- 차터 목표 자체가 프로젝트 성패를 판단하는 테스트 역할을 수행
- 기준 시간 대비 50% 처리 시간 단축 등 정량 목표는 사전/사후 측정을 요구
- 시간 박스 불충족 시 ROI 감소 등의 리스크를 명확화

### 고수준 요구 elicitation
- 전 구성원이 아이디어를 브레인스토밍 후 카드 정렬로 그룹화하여 기능 후보 도출
- 범위 외 아이디어는 추후를 위해 보관하고, 목표 달성과 직접 연관된 기능만 선택
- 기능별 상위 인수 기준을 정리해 추후 세분화 작업(스토리, 시나리오) 기반 마련

## 상세 내용

### SMART 목표 검증
- 측정 가능성 확보를 위해 사용자 설문(SUS) 등 지표 도입 사례 설명
- 달성 가능성 점검을 위해 개발·테스트 유닛이 초기 추정을 제공하고 필요 시 목표 조정

### 기능 우선순위 전략
- 비즈니스 가치, 구현 난이도, 목표 달성 기여도를 고려하여 우선순위를 정함
- Kanban, Scrum 등 다양한 일정 기법 언급하며 팀 상황에 맞는 방법 선택 제안

### 이해관계자 참여
- 스폰서, 고객, 현장 사용자, 개발·테스트가 차터 수립부터 참여해 공감대를 조성
- 테스트 관점에서 질문을 던져 모호성을 조기에 제거

## 주요 화제

### 차터와 테스트의 연결고리
- “목표 = 프로젝트 인수 테스트” 관점을 통해 전략 수준과 실행 수준을 연결

### 워크숍 퍼실리테이션
- 테스터가 중립적 퍼실리테이터 역할을 수행하며 합의 형성을 돕는 모습 소개

### 범위 관리
- 확장 아이디어를 백로그에 보존하되 현재 릴리스 목표에 집중하도록 유도

## 부차 화제

### 릴리스 미니 차터
- 각 릴리스별로 축소된 차터를 작성해 비전 대비 진행을 점검하는 기법 제안

### ROI와 사업 가치 추적
- 기능별 예상 ROI 추정을 통해 순서를 조정할 수 있음을 언급하며 경제적 의사결정 유도
