# Chapter 23: Using Tests for Other Things - 추출된 정보

## 핵심 내용
- 인수 테스트는 요구 명세 외에도 진행률 추적, 노력 추정, 스토리 분할 등 다양한 도구로 활용 가능
- 버그 보고·루트 원인 분석·회귀 검증 등 품질 활동에 테스트를 재사용하여 학습 사이클을 단축
- 테스트 통과 비율, 복잡도 등을 활용할 때는 근사치로 사용하고 맥락 해석이 필요

## 상세 핵심 내용

### 진행률 지표
- 스토리별 테스트 통과 비율을 “완료도” 추정치로 활용하되, 남은 테스트 난이도를 고려해 해석
- 스토리 완료 전 다른 작업으로 이동하는 것을 방지하기 위해 모든 테스트 통과 후에만 완료 처리

### 추정 및 분할
- 테스트 수·복잡도를 기준으로 유사 스토리와 비교해 노력 규모를 추정
- 유스케이스 시나리오, 복잡한 계산 테이블 등 테스트 단위를 기반으로 스토리를 쪼개고 병렬 개발 지원

### 버그 대응
- 결함 재현 테스트를 작성해 회귀 방지; 주변 정상 케이스를 포함해 오동작 범위 파악
- 테스트 누락·오류 원인을 분석해 요구 수집·협업 프로세스 개선

### 회귀 및 자동화
- 인수 테스트 전체를 회귀 스위트로 사용, 변경 시 기존 시나리오가 깨지면 설계/요구 변경 검토

## 주요 화제

### 분산 팀 협업
- 스토리 분할 시 서브스토리마다 인수 테스트를 정의해 책임과 종료 기준을 명확히 함

### 품질 지표 관리
- 생산 환경으로 탈출한 버그 수를 모니터링하고, 테스트 누락 이유를 프로세스 개선에 연결

## 부차 화제

### 테스트 기반 추정 한계
- 테스트 수·복잡도는 근사 지표이므로 과신하지 않도록 주의; 남은 리스크와 난이도 평가 병행

### 루트 원인 분석 실행 팁
- 지나친 분석에 빠지지 않도록 개선 가설을 빠르게 적용·검증하는 실용적 접근 제안
